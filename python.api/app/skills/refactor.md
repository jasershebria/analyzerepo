Refactor the specified code to improve quality.

Steps to follow:
1. Use todo_write to plan each refactor step as a task
2. Use rag_search and file_read to understand the current implementation
3. Apply changes with file_edit, one logical change at a time
4. Do not change behaviour — only improve structure, naming, and clarity
5. Mark each task complete as you finish it

Focus on: removing duplication, improving naming, reducing complexity, applying SOLID principles.
